## How to host on GitHub Pages

1. Create a new GitHub repository.
2. Upload all files from this folder to the **root** of the repository.
3. Go to **Settings → Pages**.
4. Under **Build and deployment**, choose:
   - **Source:** Deploy from a branch
   - **Branch:** `main`
   - **Folder:** `/ (root)`
5. Save.
6. GitHub will publish the site and give you a Pages URL.

## Important edits before you go live
You should replace:
- placeholder email addresses
- placeholder phone number
- placeholder location
- service descriptions
- about page copy
- footer disclaimer
- article titles under Research & Insights

## Connect your Namecheap domain later
After GitHub Pages is live:
1. In GitHub Pages settings, add your custom domain.
2. In Namecheap, open **Advanced DNS** for your domain.
3. Add the DNS records GitHub tells you to add.
4. Wait for DNS propagation.

## Brutal advice
Do not publish the site with fake numbers or vague claims.
A clean site with honest copy beats a pretty site with nonsense.
